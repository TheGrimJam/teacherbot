Creating a Teacherbot on the Playground

To get started, go to the Pandorabot Playground website and register for an account. Once you have done this, click on My Bots in the blue header menu and create a new Bot by clicking on Create Bot.


Once your bot is created you can start asking questions and adding responses to build a database of question and answer pairs. For a simple question-answer response you’ll need to ask your bot a question, add in your response, and click ‘Say Instead’. Next time you ask the same question your bot will respond as you have instructed.



Teaching the Teacherbot

Make your way into the files tab and click on the file labelled pand_learn. This file is your bot’s brain. As you can see the layout of a bot interaction is as follows: <category><pattern>QUESTION</pattern><template>response</template></category>
You can also add information in this section, as long as you follow the response template. Keep in mind, all questions that you ask have to be written in CAPITALS without punctuation.




You will have noticed that pand_learn is written in a programming language. This language is AIML: Artificial Intelligence Markup Language. As you continue through these tutorials you will develop a working knowledge of this language, but the two main things to remember are:

  • Tags come in pairs and should be opened and closed ( <category> </category> )

  • All your code should be placed between the <aiml> tags


Randomized Responses

This is a useful function to make your bot appear more intelligent. We can create a list of responses linked with one question, so when the bot is asked that question it can respond with an answer randomly chosen from the list. It is recommended to take a new line for each tag, to improve clarity.



Wildcards

Wildcards allow for a bit more flexibility when asking questions and recognising user input, as people can ask the same question in a hundred different ways. Wildcards act as placeholders for a set number of words.


There are 2 types of wildcards, firstly we will look at the ^ wildcard, which is a placeholder for zero or more words.



Secondly, the * wildcard, is a placeholder for one or more words. This wildcard is a bit more limited as it requires at least 1 word where you have placed the wildcard.


Synonym Sets

Sets are important because people will ask the same question in different ways, often using different words which have the same meaning, i.e synonyms. Including synonym sets means that there is a greater chance that your bot will recognise user input. In order to do this, we have to create set files. It is important to note that the text between the set tags are written in lowercase, along with the words within the set file, whilst all the remaining text is in uppercase.



Who That?

The That function is a way of creating a short conversation flow between the bot and users. It is used to continue a conversation from the previous question asked. For example:


Student: 'When is the assignment deadline?'


Teacherbot: 'Which assignment deadline?'


Student: 'The first one'


Student: 'The deadline for the first assignment is the 20th November'


In the example above Teacherbot will only respond to the question 'The first one' if its previous response was 'Which assignment deadline?. If you are adding That tags through the pand_learn file remember that all characters inside the That tag need to be uppercase and without punctuation.


Topical Punch

The Topic function introduces a topic into the conversation by grouping categories together under a common theme. This allows you to specify different answers to the same questions that depend on the Topic of the conversation.


In order to set the topic we have to use the familiar set tags as follows:
<set name = "topic">Insert Topic Name Here</set>
We can then include topic tags:
<topic name = "Insert Topic Name Here"></topic>
To add interactions within a topic we add the standard category, pattern and template tags between the topic tags.

The Illustrious Map

In order to understand maps, we first need to cover how to use the <star/> function. The <star/> is used to get the value of a * wildcard (explained above). Note the star function does not have an opening and closing tag as most others do, it is implemented with a single <star/>.




Now to tackle Maps, Maps are name-value pairs which act as a database. For example we could build a Map of assignment deadlines, with the name of an assignment and its associated deadline/value.


The video below shows the creation of an assignment deadlines map. In the category, the name of the map file must be specified using: <map name = "Insert Map File Name Here">.
The <star/> tag is then placed in between the map tags. The star value acts as the name, and the map file is searched to find a corresponding value; in our case the deadline date.



Recursion

Don't be intimidated by the name, recursion is a simple and straightforward function. It can be used to replace colloquial expressions and correct spelling errors, making the bot adaptable.


Recursion is implemented through the <srai> tags, allowing you to reuse the answer from a previous question. As is shown below, the <srai> tags are placed in the <template> tags redirecting to the associated response.
